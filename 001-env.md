## Setting Up SSH Connectivity

Passwordless SSH between servers is required to run remote commands easily. Passwordless SSH depends on:

- The `sshd` daemon having `PubkeyAuthentication` enabled (in `/etc/ssh/sshd_config` or `/etc/ssh/sshd_config.d`)
- The current user having an SSH key pair whose public key is recorded in the remote server's `~/.ssh/authorized_keys` — use `ssh-copy-id` to do this
- The remote host signature being registered in `~/.ssh/known_hosts`

The script [scripts/1-1-setupssh.sh](scripts/1-1-setupssh.sh) handles all of this, provided you can log in to the remote MQ servers as `root` using a password.

## Setting Up HAProxy

A load balancer is used to route client requests to the active Queue Manager. This environment uses HAProxy, which is easy to install on the bastion node.

HAProxy and its configuration can be set up using [scripts/1-2-haproxy.sh](scripts/1-2-haproxy.sh). To configure it manually, follow these steps:

1. Install HAProxy using the `dnf` command:

    ``` bash
    sudo dnf install -y haproxy
    ```

2. Back up the HAProxy configuration file:

    ``` bash
    sudo cp /etc/haproxy/haproxy.cfg /etc/haproxy/haproxy.cfg.bak
    ```

3. Create a new HAProxy configuration as follows, substituting the variables with the appropriate hostnames and IP addresses:

    ```
    defaults
        mode                    tcp
        log                     global
        option                  tcplog
        option                  tcp-check
        timeout connect         500ms
        timeout client          50000ms
        timeout server          50000ms
        timeout check           20ms

    #---------------------------------------------------------------------
    # main frontend which proxies to the backends
    #---------------------------------------------------------------------
    frontend mqlistener
        bind *:1414
        mode tcp
        option tcplog
        default_backend             myqmgr

    #---------------------------------------------------------------------
    # round robin balancing between the various backends
    #---------------------------------------------------------------------
    backend myqmgr
        balance roundrobin
        option tcp-check
        tcp-check connect port 1414

        server  ${host11} ${ip11}:1414 check 
        server  ${host12} ${ip12}:1414 check 
        server  ${host13} ${ip13}:1414 check 
        server  ${host21} ${ip21}:1414 check 
        server  ${host22} ${ip22}:1414 check 
        server  ${host23} ${ip23}:1414 check 
    ```

4. Start the HAProxy service:

    ``` bash
    sudo systemctl start haproxy
    ```

5. Verify that the HAProxy service started successfully:

    ``` bash
    sudo systemctl status haproxy
    sudo journalctl -u haproxy
    netstat -an | grep 1414 
    ```

## Enabling GUI for Bastion

> **Note:** This section is optional.

For those who want a graphical interface, use the script [scripts/1-3-installxfce.sh](scripts/1-3-installxfce.sh) to install the Xfce window manager and Firefox browser. This is also useful for accessing the MQ Web Console.

To install step by step, follow this procedure:

1. Install Xfce and TigerVNC server:

    ``` bash
    sudo dnf -y install tigervnc-server
    sudo dnf -y install https://dl.fedoraproject.org/pub/epel/epel-release-latest-9.noarch.rpm
    sudo dnf --enablerepo=epel group
    sudo dnf -y groupinstall "Xfce"
    sudo systemctl set-default graphical.target
    ```

2. Configure the VNC server:

    1. Set the password using the `vncpasswd` command.
    2. Enable the first VNC server instance: `systemctl enable vncserver@:1`
    3. Edit `/etc/tigervnc/vncserver.users` and add:

        ```
        :1=root
        ```

    4. Edit `/etc/tigervnc/vncserver-config-defaults` to use Xfce as the session:

        ``` bash
        session=xfce
        ```

    5. Start the VNC server: `systemctl start vncserver@:1`

3. Xfce does not include a web browser. Install Firefox with:

    ``` bash
    sudo dnf makecache --refresh
    sudo dnf install firefox
    ```

4. Connect a VNC client to port 5901 on the server. To use an SSH tunnel instead, run:

    ``` bash
    ssh -L 5901:localhost:5901 <load balancer host>
    ```

## Installing MQ Client

Download the MQ Client from IBM Fix Central.

Run the script [scripts/1-4-mqclient.sh](scripts/1-4-mqclient.sh) as root on the load balancer host, or follow these manual steps:

1. Extract and install the MQ client packages:

    ``` bash
    tar -xvzf 10.0.0.0-IBM-MQC-LinuxX64.tar.gz
    cd MQClient
    ./mqlicense.sh -accept
    dnf install -y MQ*
    ```

2. Set up the environment variables in the system profile:

    ``` bash
    echo "source /opt/mqm/bin/setmqenv -s" > /etc/profile.d/mqm.sh
    echo "export PATH=/opt/mqm/samp/bin:$PATH" >> /etc/profile.d/mqm.sh
    chmod 755 /etc/profile.d/mqm.sh
    ```
