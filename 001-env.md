# Environment preparation

The environment is run with 7 VMs running RHEL 9. The first 3 is running the first site and then another 3 running as the backup site; one VM running as the load balancer and bastion node. 

The configuration is as follows:

The setup is using a minimal 2CPU 8GB RAM for the MQ servers and the bastion node (load balancer). The following steps must be run on the load balancer (bastion) node as root. 
Clone this git repository into the bastion/load balancer node:

``` bash
git clone https://github.com/vbudi000/mq-ha-crr.git
```

Edit the file `mq-ha-crr/scripts/hacrrenv.sh` to change the hostnames for the Queue Manager hosts.

## Setting up SSH connectivity

To make it easy to run a remote commands between server, a passwordless connectivity to execute commands is necessary. Passwordless ssh session is determined by:
- The sshd supports PubkeyAuthentication (in `/etc/ssh/sshd_config` or in `/etc/ssh/sshd_config.d`) 
- The current user has a SSH keypair on which its public key is recorded on the remote server's `~/.ssh/authorized_keys` use the `ssh-copy-id`
- The remote host signature is registered in the `~/.ssh/known_hosts` 

The script in [scripts/1-1-setupssh.sh](scripts/1-1-setupssh.sh) allows you to do that - as long as you can login to the remote MQ servers using a password. 

## Setting up haproxy 

We are using a load balancer to route request from a client. We can also use a CCDT table later on, but some configuration do need a load balancer. In this scenario, we are using haproxy that is easily installed in the bastion node. 

The haproxy and its configuration can be configured using the process similar to [scripts/1-2-haproxy.sh](scripts/1-2-haproxy.sh). If you choose to run the commands manually - here are the steps:

1. Install haproxy using the `dnf` command:

    ``` bash
    sudo dnf install -y haproxy
    ```

2. Backup the haproxy configuration file /etc/haproxy/haproxy.cfg. 

    ``` bash
    sudo cp /etc/haproxy/haproxy.cfg /etc/haproxy/haproxy.cfg.bak
    ```

3. Create a new haproxy configuration as follows, substitue the variables with the appropriate hostnames and IP addresses:

    ```
    defaults
        mode                    tcp
        log                     global
        option                  tcplog
        option                  tcp-check
        timeout connect         500ms
        timeout client          50000ms
        timeout server          50000ms
        timeout check           20ms

    #---------------------------------------------------------------------
    # main frontend which proxys to the backends
    #---------------------------------------------------------------------
    frontend mqlistener
        bind *:1414
        mode tcp
        option tcplog
        default_backend             myqmgr

    #---------------------------------------------------------------------
    # round robin balancing between the various backends
    #---------------------------------------------------------------------
    backend myqmgr
        balance roundrobin
        option tcp-check
        tcp-check connect port 1414

        server  ${host11} ${ip11}:1414 check 
        server  ${host12} ${ip12}:1414 check 
        server  ${host13} ${ip13}:1414 check 
        server  ${host21} ${ip21}:1414 check 
        server  ${host22} ${ip22}:1414 check 
        server  ${host23} ${ip23}:1414 check 
    ```
4. Start the haproxy service

    ``` bash
    sudo systemctl start haprxy
    ```

5. Verify that the haproxy service is started successfully

    ``` bash
    sudo systemctl status haproxy
    sudo journalctl -u haproxy
    netstat -an | grep 1414 
    ```

## Enabling GUI for bastion (optional - to use MQ Web)

For those who want have a Graphical interface, you can use the script in [scripts/1-3-instalxfce.sh](scripts/1-3-installcfce.sh) to start Xfce X-Windows manager and install the Firefox browser. This is also useful for using the MQ Web server to browse the Queue Manager resources.

To perform the installation step by step, use the following procedure:

1. Install xfce and tiger vnc server:

    ``` bash
    sudo dnf -y install tigervnc-server
    sudo dnf -y install https://dl.fedoraproject.org/pub/epel/epel-release-latest-9.noarch.rpm
    sudo dnf --enablerepo=epel group
    sudo dnf -y groupinstall "Xfce"
    sudo systemctl set-default graphical.target
    ```

2. Configure the VNC server:

    1. Set the password using `vncpasswd` command
    2. Run the `vncserver` command to initialize VNC server
    3. Run `vncserver -list` to list the session and then kill using `vncserver -kill <session>` command
    4. Edit the file `~/.vnc/xstartup` to contain the following:

        ``` bash
        #!/bin/sh
        unset SESSION_MANAGER
        unset DBUS_SESSION_BUS_ADDRESS
        xrdb $HOME/.Xresources
        startxfce4 &
        ```
    5. Now start VNC server again using `vncserver` command

3. The Xfce does not come with a Web browser, you can install Firefox using the following commands:

    ``` bash
    sudo dnf config-manager addrepo --id=mozilla --set=baseurl=https://packages.mozilla.org/rpm/firefox --set=gpgkey=https://packages.mozilla.org/rpm/firefox/signing-key.gpg --set=gpgcheck=1 --set=repo_gpgcheck=0 --set=priority=10
    sudo dnf makecache --refresh
    sudo dnf install firefox
    ```

4. You can now run VNC client to port 5901 on the server - or if you want to use SSH tunnel, you can run the following command:

    ``` bash
    ssh -L 5901:localhost:5901 <load balancer host>
    ```
